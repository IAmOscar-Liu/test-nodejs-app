    <abstract-dtext>
      <p general="">A financial service system includes an automated teller machine and a server. The automated teller machine is configured to accept a debit card. The server is configured to execute an authentication procedure after accepting an activation request related to the debit card. The server is further configured to send a default personal identification number (PIN) information for the debit card to an electronic device related to the debit card. The server is further configured to activate the debit card as the automated teller machine accepts the default PIN information for the debit card.</p>

      <p general="0001">本揭露係有關於一種金融服務系統，尤其是用於金融卡開卡的一種金融服務系統。</p>

    <background-art>
      <p general="0002">在金融卡新卡的啟用程序中，持卡人必須臨櫃或至自動櫃員機完成卡片基碼的變更作業，方可使用金融卡的功能。</p>
      <p general="0003">然而，紙本印製的基碼通知書可能因遺失或遭惡意第三人竊取利用，產生卡片盜用而致存款損失之風險。又紙本之基碼通知書由銀行印製完成後，須經由行內傳遞、郵寄予持卡人等程序，均可能造成個人或機敏資料外洩之風險。</p>
      <p general="0004">因此，如何提出一種可解決上述問題的金融服務系統，是目前業界亟欲投入研發資源解決的問題之一。</p>

    <disclosure>
      <p general="0005">有鑑於此，本揭露之一目的在於提出一種可有解決上述問題的金融服務系統。</p>
      <p general="0006">為了達到上述目的，依據本揭露之一實施方式，一種金融服務系統包含自動櫃員機以及伺服器。自動櫃員機配置以接收金融卡。伺服器配置以在自動櫃員機接受關於金融卡之開卡請求之後，執行身份認證程序。伺服器配置以在身份認證程序通過之後，發送金融卡基碼資料至與金融卡相關連之電子裝置。伺服器配置以在自動櫃員機接受金融卡基碼資料時，將金融卡開卡。</p>
      <p general="0007">於本揭露的一或多個實施方式中，自動櫃員機進一步包含輸入單元以及顯示單元。輸入單元配置以接收開卡請求、輸入資料以及金融卡基碼資料。顯示單元配置以顯示使用者介面。</p>
      <p general="0008">於本揭露的一或多個實施方式中，伺服器進一步配置以在自動櫃員機接受開卡請求且確認電子裝置已與金融卡相關連之帳戶資訊綁定之後，執行身份認證程序。</p>
      <p general="0009">於本揭露的一或多個實施方式中，伺服器進一步配置以在自動櫃員機接受開卡請求之後，確認與金融卡相關連之行動網銀用戶是否登入，以及在確認行動網銀用戶登入之後，確認電子裝置是否與帳戶資訊綁定。</p>
      <p general="0010">於本揭露的一或多個實施方式中，伺服器係以推播方式將金融卡基碼資料發送至電子裝置。</p>
      <p general="0011">於本揭露的一或多個實施方式中，身份認證程序係使自動櫃員機之顯示單元顯示條碼資訊，且伺服器係在經由電子裝置接收到條碼資訊時認定身份認證程序通過。</p>
      <p general="0012">於本揭露的一或多個實施方式中，身份認證程序係在自動櫃員機接收到身份認證請求時發送驗證碼至電子裝置，且伺服器係在經由自動櫃員機接收到驗證碼時認定身份認證程序通過。</p>
      <p general="0013">於本揭露的一或多個實施方式中，身份認證程序係使自動櫃員機之顯示單元顯示身份資訊輸入畫面，當伺服器係經由自動櫃員機接收到之輸入資料與金融卡相關連之帳戶資訊匹配時發送驗證碼至電子裝置，且伺服器係在經由自動櫃員機接收到驗證碼時認定身份認證程序通過。</p>
      <p general="0014">於本揭露的一或多個實施方式中，其中該伺服器進一步配置以在自動櫃員機接受開卡請求之後，確認與金融卡相關連之行動網銀用戶是否登入，以及在確認行動網銀用戶未登入之後，使自動櫃員機之顯示單元顯示身份資訊輸入畫面。</p>
      <p general="0015">於本揭露的一或多個實施方式中，伺服器係以簡訊方式將金融卡基碼資料發送至電子裝置。</p>
      <p general="0016">綜上所述，於本揭露的金融服務系統中，規劃了金融卡基碼推播以及客戶身分確認的機制，透過行動設備或簡訊認證持卡人身份後，再以裝置推播或簡訊發送金融卡基碼至持卡人的電子裝置。本揭露不但能防止因遺失或遭惡意第三人竊取利用，產生卡片盜用而致存款損失之風險，還能避免個人或機敏資料外洩之風險，兼顧了客戶的便利性以及風險控管。</p>

      <p general="0019">以下將以圖式揭露本揭露之複數個實施方式，為明確說明起見，許多實務上的細節將在以下敘述中一併說明。然而，應瞭解到，這些實務上的細節不應用以限制本揭露。也就是說，在本揭露部分實施方式中，這些實務上的細節是非必要的。此外，為簡化圖式起見，一些習知慣用的結構與元件在圖式中將以簡單示意的方式繪示之。在所有圖式中相同的標號將用於表示相同或相似的元件。</p>
      <p general="0020">本揭露提到「與金融卡相關連」之用語，意指任一元件包含金融卡相關之資訊，或意指任一元件與該金融卡綁定。</p>
      <p general="0021">本揭露提到「匹配」之用語，意指多個資訊相互比對吻合。</p>
      <p general="0022">以下將詳細介紹本實施方式之金融服務系統100所包含的各元件的結構、功能以及各元件之間的連接關係。</p>
      <p general="0023">請參照第1圖，其為繪示根據本揭露一實施方式之金融服務系統100的功能方塊圖。如第1圖所示，本實施方式之金融服務系統100包含自動櫃員機110以及伺服器120。伺服器120實體或通訊連接自動櫃員機110。自動櫃員機110配置以接受金融卡以及金融卡開卡請求。伺服器120配置以在自動櫃員機110接受關於金融卡之開卡請求之後執行身分認證。伺服器120還配置以在身份認證程序通過之後發送金融卡基碼資料至與金融卡相關連之電子裝置200。伺服器120還配置以在自動櫃員機110接受金融卡基碼資料時將金融卡開卡。</p>
      <p general="0024">具體來說，自動櫃員機110包含輸入單元112以及顯示單元114。輸入單元112配置以接收開卡請求、輸入資料以及金融卡基碼資料。顯示單元114配置以顯示使用者介面。伺服器120包含資料庫122、驗證單元124、推播單元126以及通訊介面128。資料庫122配置以儲存帳戶資訊、金融卡基碼資料、條碼資訊以及驗證碼資訊。驗證單元124配置以執行身分認證程序。推播單元126配置以推播資訊以及發送簡訊至與金融卡相關連之電子裝置200。通訊介面128配置以使伺服器120通訊連接自動櫃員機110以及電子裝置200。</p>
      <p general="0025">藉由前述結構配置，當持卡人將金融卡插入自動櫃員機110之插卡口時，自動櫃員機110的輸入單元112可接受持卡人輸入的開卡請求，隨後伺服器120的驗證單元124執行身分認證程序。當身分認證程序通過之後，推播單元126推播金融卡基碼資料至與金融卡相關連之電子裝置200。當自動櫃員機110接受持卡人於輸入單元112輸入之金融卡基碼資料時，伺服器120將金融卡開卡。</p>
      <p general="0026">請參照第2圖，其為繪示第1圖中之金融服務系統100的金融卡開卡的流程圖。</p>
      <p general="0027">在本實施方式中，在執行步驟S205之前，持卡人需先於銀行臨櫃辦理創建銀行帳戶。持卡人可選擇成為行動網銀應用程式(以下簡稱APP)用戶，並可選擇辦理電子裝置200與金融卡帳戶資訊綁定。在步驟S205中，於持卡人將金融卡插入自動櫃員機110的插卡口時，自動櫃員機110可經由輸入單元112接受持卡人輸入的開卡請求(例如，點選使用者介面上顯示的「金融卡開卡」選項或按壓與「金融卡開卡」選項對應的按鍵)。在自動櫃員機110接受開卡請求後，伺服器120首先判斷持卡人是否於電子裝置200登入行動網銀APP。若持卡人成功登入行動網銀APP，且再點選行動網銀APP畫面中的「金融卡開卡」選項時，伺服器120再判斷電子裝置200是否確實與資料庫122中之與金融卡相關連的帳戶資料綁定。</p>
      <p general="0028">在步驟S210中，若伺服器120判斷電子裝置200已經與帳戶資料綁定，伺服器120將執行身分認證程序。身分認證程序係使自動櫃員機110的顯示單元114顯示條碼資訊(例如，二維條碼(QR Code))。電子裝置200對此條碼資訊掃描後，電子裝置200將經由行動網銀APP傳送此條碼資訊至伺服器120的驗證單元124執行驗證。當驗證單元124接收到此條碼資訊後，隨即認定身分認證程序通過。</p>
      <p general="0029">在步驟S215中，當驗證單元124認定身分認證程序通過後，伺服器120的推播單元126以推播方式將金融卡基碼資料發送至電子裝置200，並呈現於行動網銀APP的顯示畫面中。</p>
      <p general="0030">在步驟S220中，持卡人根據顯示於電子裝置200的金融卡基碼資料於自動櫃員機110的輸入單元112輸入此金融卡基碼資料。</p>
      <p general="0031">在步驟S225中，當自動櫃員機110接受經由輸入單元112輸入的金融卡基碼資料後，隨即傳送此金融卡基碼至伺服器120的驗證單元124執行驗證，驗證通過後伺服器120將金融卡開卡，以完成金融卡開卡流程。</p>
      <p general="0032">在一些實施方式中，在步驟S210中，若電子裝置200無法對條碼資訊進行掃描，持卡人可於輸入單元112輸入身分認證請求(例如，點選使用者介面上顯示的「輸入驗證碼」選項或按壓與「輸入驗證碼」選項對應的按鍵)。當自動櫃員機110接受身分認證請求後，自動櫃員機110將此身分認證請求傳送至伺服器120，伺服器120的推播單元126再以推播方式將驗證碼發送驗證碼至電子裝置200，並呈現於行動網銀APP的顯示畫面中。持卡人根據顯示於電子裝置200的驗證碼於自動櫃員機110的輸入單元112輸入此驗證碼。自動櫃員機110將傳送此驗證碼至伺服器120的驗證單元124執行驗證。當驗證單元124判斷驗證碼比對成功，隨即認定身分認證程序通過。隨後進入上述之步驟S215、S220以及S225以完成金融卡開卡流程。</p>
      <p general="0033">在一些實施方式中，在步驟S205中，若持卡人未於銀行臨櫃辦理成為行動網銀用戶，持卡人將金融卡插入自動櫃員機110的插卡口，持卡人於輸入單元112輸入開卡請求。開卡請求包含持卡人點選使用者介面上顯示的「金融卡開卡」選項或按壓與「金融卡開卡」選項對應的按鍵，以及持卡人於使用者介面上點選「非行動網銀用戶」選項或按壓與「非行動網銀用戶」選項對應的按鍵。</p>
      <p general="0034">在步驟S210中，當自動櫃員機110接受此開卡請求，隨後伺服器120將執行身分認證程序。身分認證程序係使自動櫃員機110的顯示單元114顯示身分資訊輸入畫面。舉例來說，此身分認證輸入畫面首先要求輸入手機號碼。在持卡人於輸入單元112輸入手機號碼資訊後，自動櫃員機110將手機號碼資訊傳送至伺服器120的驗證單元124執行驗證。當驗證單元124判斷此手機號碼資訊與儲存於資料庫122中與金融卡相關連之帳戶資訊匹配時，伺服器120繼續執行身分認證程序。</p>
      <p general="0035">在此的身分認證程序例如使自動櫃員機110的顯示單元114顯示另一身分資訊輸入畫面。舉例來說，此身分資訊輸入畫面要求輸入基本資料驗證。在持卡人於輸入單元112輸入基本資料驗證後，自動櫃員機110將基本資料驗證傳送至伺服器120的驗證單元124執行驗證。當驗證單元124判斷此基本資料驗證與儲存於資料庫122中與金融卡相關連之帳戶資訊匹配時，伺服器120的推播單元126再以簡訊方式將驗證碼發送驗證碼至電子裝置200。持卡人根據顯示於電子裝置200的驗證碼於自動櫃員機110的輸入單元112輸入此驗證碼。自動櫃員機110將傳送此驗證碼至伺服器120的驗證單元124執行驗證。當驗證單元124判斷驗證碼比對成功，隨即認定身分認證程序通過。</p>
      <p general="0036">在步驟S215中，當驗證單元124認定身分認證程序通過後，伺服器120的推播單元126以簡訊方式將金融卡基碼資料發送至電子裝置200。隨後進入上述之步驟S220以及S225以完成金融卡開卡流程。</p>
      <p general="0037">由以上對於本揭露之具體實施方式之詳述，可以明顯地看出，在本新型的金融服務系統中，藉由包含條碼及/或驗證碼或基本資料驗證的身分認證程序，以避免因遺失或遭惡意第三人竊取利用個資的風險。在本新型的金融服務系統中，藉由推播機制或簡訊發送金融卡基碼資料至電子裝置，除了防止金融卡基碼資料遺失或遭惡意利用的風險，更能提升用戶使用的便利性。</p>
